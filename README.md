# 🐺 狼人殺輔助器 Werewolf Assistant

線下狼人殺的網頁輔助工具。支援 **8～16 人**，提供 **語音引導** 與 **規則防呆機制**，
讓桌邊不需要實體法官也能順暢進行。

## ✨ 功能特色

- **自由配置**：自選遊玩人數（8–16）與要使用的角色，內建各人數推薦板子。
- **兩種模式**
  - 🎮 **遊玩模式**：建立 4 位數房號，其他人用手機掃 QR 或輸入房號加入。系統自動：
    抽號碼 → 私下發角色（狼人互相可見隊友）→ 語音引導每個夜晚 / 白天流程。
  - 🧑‍⚖️ **法官模式**：單機輔助。你就是法官，網頁負責語音提示、計時、記錄場上狀態與結算。
    完全在瀏覽器執行，可離線使用。開局有三種發牌方式：
    - **系統發牌＋傳手機看角色**：把手機依號碼順序傳給每位玩家，各自蓋牌看自己的角色。
    - **系統發牌＋直接開始**：法官查看角色分配表。
    - **實體牌＋第一晚邊進行邊登記**：玩家拿實體牌，第一晚每個角色起身時由法官登記是誰並當場行動（登記守衛→守衛守人、登記狼人→狼人刀人…），沒被登記到的座位自動視為平民。
  - 每個操作步驟都有「← 上一步」可復原，避免法官按錯（含二次確認）。
- **語音引導**：使用瀏覽器內建語音（zh-TW），免外部服務。可隨時開關。
- **防呆機制**（重點）
  - 🧪 **女巫第一晚不能自救**：若女巫第一晚被刀並嘗試用解藥救自己，系統會擋下並顯示「第一晚女巫不能自救！」。（可在設定改為「全程不能自救」或「皆可自救」）
  - 🧪 女巫同一晚預設不能同時用解藥與毒藥（可開關）。
  - 🛡️ **守衛不能連續兩晚守護同一人**。
  - 🛡️🧪 **同守同救**：守衛與女巫同時作用於同一人 → 該玩家視為死亡。
  - 🔫 **獵人 / 狼王被女巫毒死時無法開槍**。
  - 🤪 **白痴**被投票時翻牌不死、失去投票權；被刀或被毒則正常死亡。

## 🎭 支援角色

| 角色 | 陣營 | 能力 |
|------|------|------|
| 🐺 狼人 | 狼 | 每晚與同伴擊殺一人 |
| 👑 狼王 | 狼 | 出局時可開槍（被毒除外） |
| 🐉 白狼王 | 狼 | 可在白天自爆並帶走一人，自爆後直接天黑；被殺/被票不能開槍 |
| 🌑 隱狼 | 狼 | 夜晚不睜眼刀人，預言家查驗顯示為好人 |
| 👨‍🌾 平民 | 民 | 無技能，靠推理投票 |
| 🔮 預言家 | 神 | 每晚查驗一人好 / 壞 |
| 🧪 女巫 | 神 | 解藥、毒藥各一瓶 |
| 🔫 獵人 | 神 | 出局時可開槍（被毒除外） |
| 🛡️ 守衛 | 神 | 每晚守護一人免疫狼刀（不可連守） |
| 🤪 白痴 | 神 | 被票出翻牌不死，失去投票權 |
| ⚔️ 騎士 | 神 | 白天可與一人決鬥：對方是狼→對方死且天黑；對方是好人→騎士死 |
| 🧓 長老 | 神 | 可扛過第一次狼刀（毒、投票不適用） |
| 💘 邱比特 | 神 | 第一晚指定兩名戀人，任一人死亡另一人殉情；異營戀人若為最後兩人則戀人勝 |

夜晚行動順序：**守衛 → 狼人 → 女巫 → 預言家**。
勝利條件可選 **屠邊**（神全滅或民全滅即狼勝）或 **屠城**（好人全部出局狼才勝）。

## 🚀 執行方式

```bash
npm install
npm start          # 啟動於 http://localhost:3000
```

- 房主 / 法官開啟首頁，設定人數、角色與規則後選擇模式。
- 遊玩模式下，其他玩家用手機開啟 `http://<你的IP>:3000/player.html` 或掃描房號 QR 碼加入
  （手機需與主機在同一區網）。

## 🌐 部署到 GitHub Pages

本專案已內建 GitHub Actions 工作流程（`.github/workflows/deploy-pages.yml`），
會把 `public/` 資料夾自動發佈到 GitHub Pages。

**首次設定（只需一次）**：
1. 進入 GitHub 儲存庫 → **Settings** → **Pages**。
2. **Build and deployment → Source** 選擇 **GitHub Actions**。
3. 推送到 `main` 或 `claude/werewolf-game-webapp-fEpYO` 分支即會自動部署，
   也可在 **Actions** 分頁手動執行 *Deploy to GitHub Pages*。
4. 完成後網址為 `https://<你的帳號>.github.io/werewolf/`。

> 注意：GitHub Pages 只提供靜態託管。
> - 🧑‍⚖️ **法官模式**：完整可用（純前端，可離線）。
> - 🎮 **遊玩模式**：需要 Node 後端（Socket.IO）。在 Pages 上開啟遊玩模式會顯示說明，
>   你可以把後端網址填入，或在加入網址後加上 `?server=<後端網址>`。

### 讓遊玩模式也能線上連線（選用）

把整個專案部署到任一支援 Node 的平台（Render / Railway / Fly.io 等），啟動
`npm start`，取得後端網址後：
- 房主在 Pages 的遊玩模式畫面填入該網址；玩家加入網址會自動帶上 `?server=`。
- 或直接用該後端網址本身當作網站入口（後端同時提供前端與 Socket.IO）。

### 在自己的伺服器執行並自動更新

```bash
git clone https://github.com/morningfly7012/werewolf.git
cd werewolf && npm install
sudo npm install -g pm2
pm2 start npm --name werewolf -- start   # 背景常駐
pm2 save && pm2 startup                   # 開機自動啟動
```

讓伺服器**自動跟上最新版本**（每 5 分鐘檢查一次 `main`，有更新就拉取並重啟）：

```bash
chmod +x scripts/auto-update.sh
crontab -e
# 加入這一行（路徑換成你的實際路徑）：
*/5 * * * * /path/to/werewolf/scripts/auto-update.sh >> /tmp/werewolf-update.log 2>&1
```

`scripts/auto-update.sh` 會比對遠端 `main`，有新 commit 才 `git reset --hard` + `npm install` + `pm2 restart werewolf`。
想要「推送後立即更新」可改用 GitHub Webhook 觸發同一支腳本（需對外開一個接收端點）。

## 🗂️ 專案結構

```
server/
  index.js          Express + Socket.IO 伺服器、房間事件
  game/
    roles.js        角色定義、推薦板子、配置驗證
    engine.js       規則引擎與所有防呆判定（前後端共用）
    rooms.js        房間管理
public/
  index.html        首頁：設定與選模式
  host.html         遊玩模式主畫面（房號、語音引導）
  player.html       玩家手機端（抽號碼、看角色、夜晚行動）
  judge.html        法官模式（瀏覽器直接執行引擎）
  join.html         以房號加入的轉址頁
  js/               voice.js（語音）、ui.js（共用元件）
  css/styles.css
```

規則引擎 `engine.js` / `roles.js` 為純 ES 模組，後端與法官模式前端共用同一份邏輯，
確保兩種模式行為一致。
